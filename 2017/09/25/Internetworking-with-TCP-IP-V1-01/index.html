Internetworking with TCP/IP V1 01

2017-09-25

Recently, i’m reading Douglas E.Comer’s classical books: Internetworking with TCP/IP. This series of books covers almost all fields of networking.

Now I’m reading on Chapter 17. The previous chapters talk about basic network acknowledges, including network basic, network layers, basic TCP/IP protocols , route protocols and so on. Chapter 17 talks about Packet Classification. It’s relatively new, so I want to write some notes.

First of all, traditional networks use layering technique to transfer packets. However, it doesn’t meet all transfer requirements, such as MPLS(Multi-Protocol Label Switching). So, a new mechanism called classification is invented.

A classifier treats each packet as an array of bits and checks the contents of fields at specific locations in the array.

We can regard classification as a layer between Interface layer and IP layer. It checks a packet, if any of the classification rules match the packet, the classification layer follow the rules. Otherwise, pass the packet up to the traditional protocol stack. Here is a figure showing the structure:

The most significant advantage of a classification mechanism maybe the generalizations it enables.

Classification is used woth MPLS as well as in Ethernet switches. Most Ethernet switches implement classification in hardware. A hardware technology known as TCAM uses parallelism to perform classification at extremely high speed.
